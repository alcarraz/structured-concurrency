\section{Programación reactiva}\label{sec:el-problema:-reactive-programming}

\begin{frame}
    \frametitle{Problemas del Enfoque Reactivo}
    \begin{itemize} [<+->]
        \item \textbf{Legibilidad:} Callback Hell $\rightarrow ~$ Lógica anidada difícil de seguir
        \item \textbf{Manejo de recursos:} No es trivial garantizar su liberación.
        \item \textbf{Excepciones:} Trazas del stack confusas, pueden saltar en el framework en lugar del código que ls genera
        \item \textbf{Carga cognitiva:} Alto costo mental para entender el código
        \item \textbf{Mantenibilidad:} Lógica de negocio mezclada con la lógica de control.
        \item \textbf{Depuración:} Difícil de seguir la secuencia de instrucciones.
    \end{itemize}
